# *Nuitka Command GUI*

***Nuitka Command GUI*** 是一款专为 Python 开发者设计的 Nuitka 可视化打包配置工具。它将复杂的命令行参数转化为直观的图形界面，支持从基础打包到深层性能优化的全方位配置，旨在大幅降低 Nuitka 的使用门槛。

## 🚀 核心特性

- **全模式支持**：一键切换 **.exe (窗口/控制台)** 或 **.pyd (加密模块)** 编译模式。
- **深层适配**：
  - **Qt 框架精细化**：独立勾选 PyQt5、PySide2 或 PySide6。
  - **插件系统**：内置常用插件开关（Numpy, Torch, Tkinter 等），支持**自定义插件名称**手动输入。
- **性能榨汁机**：支持 LTO 优化选择、多线程并行编译控制（支持手动开关锁定）。
- **Windows 增强**：完整支持启动画面 (Splash Screen)、图标嵌入、版本号/版权/商标等资源注入。
- **高级调试**：支持标准输出 (stdout) 和标准错误 (stderr) 实时定向到外部日志文件。
- **数据管理**：可视化管理包数据、外部目录和文件的包含与排除逻辑。



## 🛠️ 安装要求

1. **Python 环境**：建议使用 Python 3.8+

2. **依赖库安装**：

   ```bash
   pip install ttkthemes
   ```

3. **打包引擎**：确保系统已安装 [Nuitka](https://nuitka.net/) 及其对应的 C 编译器（如 MinGW64 或 MSVC）。

## 📖 使用指南

### 1. 基础打包 (Standalone)

1. 在“核心输出路径”中选择你的入口 `.py` 文件。
2. 在“编译模式”中选择 `窗口程序`。
3. 勾选 `独立分发模式 (--standalone)`。
4. 点击 **生成 Nuitka 命令** 并点击 **复制**，在终端粘贴运行。

### 2. 性能优化技巧

- **并行编译**：在“性能参数”中勾选“指定编译线程”，填入你的 CPU 核心数，可缩短 50% 以上的编译时间。
- **LTO (Link Time Optimization)**：建议设为 `yes` 或 `thin`，这能让生成的 exe 运行效率更高。

### 3. 处理外部资源

- 如果你的代码引用了当前目录下的 `config` 文件夹，请在“插件/数据管理”的 **包含目录** 中填入：`config=config`。

## ⚠️ 注意事项

- **Qt 插件冲突**：请确保你的项目中只勾选了一个 Qt 框架（例如只选 PySide6），同时选中多个可能导致打包失败。
- **单文件模式**：使用 `onefile` 模式时，建议配合 `启动画面` 使用，否则大型程序加载时的“静默期”会让用户误以为启动失败。
- **路径空格**：工具已自动处理 Windows 路径中的空格（加引号），但仍建议工程路径尽量使用英文且不带特殊字符。
